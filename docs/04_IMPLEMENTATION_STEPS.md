# Project 1 — Technical Implementation Steps

## Complete Step-by-Step for Platform Preparation

This document outlines every technical step needed to take the AI Safety Lab
from code to operational deployment on the DGX Spark cluster.

---

## Phase 1: Environment Setup (Week 1)

### Step 1.1: DGX Partition Access
- [ ] Obtain SSH credentials for team partition
- [ ] Verify GPU availability: `nvidia-smi`
- [ ] Confirm Python 3.11+: `python3 --version`
- [ ] Confirm disk space: `df -h` (need 20+ GB)

### Step 1.2: Ollama Installation
- [ ] Install Ollama: `curl -fsSL https://ollama.com/install.sh | sh`
- [ ] Start server: `OLLAMA_HOST=0.0.0.0:11434 ollama serve &`
- [ ] Verify: `curl http://localhost:11434/api/tags`

### Step 1.3: Model Downloads
- [ ] Pull Llama Guard 3 (8B): `ollama pull llama-guard3:8b` (~5 GB)
- [ ] Pull Mistral 7B: `ollama pull mistral:7b-instruct` (~5 GB)
- [ ] Pull Llama Guard Vision: `ollama pull llama-guard3:11b-vision` (~7 GB)
- [ ] Verify all loaded: `ollama list`

### Step 1.4: Project Setup
- [ ] Clone project repo to DGX
- [ ] Install dependencies: `pip install -r requirements.txt`
- [ ] Run health check: `python -m config.health_check`
- [ ] Confirm all 6 checks pass

## Interfaces with GitHub Repositories

The system interfaces with GitHub repos at these points:

1. **Cloning** (`ingestion/github_loader.py`):
   - `git clone --depth 1 <url>` into `cloned_repos/`
   - Repo profiling: language detection, entry point discovery

2. **Dependency Installation** (`ingestion/sandbox_runner.py`):
   - Reads target repo's `requirements.txt`
   - Installs via pip in sandboxed context
   - Handles missing requirements.txt gracefully

3. **Execution** (`ingestion/sandbox_runner.py`):
   - Runs target model's entry point via subprocess
   - Pipes test prompt via stdin
   - Captures stdout/stderr/exit code

4. **Output Collection** (`ingestion/output_capture.py`):
   - Parses text responses (plain text and JSON)
   - Collects generated files (images, reports)
   - Detects refusals and error states

5. **Cleanup** (not automatic):
   - Cloned repos persist in `cloned_repos/` for re-evaluation
   - Manual cleanup: `rm -rf cloned_repos/*`
